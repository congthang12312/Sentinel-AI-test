# Coding Level 1: Junior Developer

**Target Audience**: Learning developers, bootcamp graduates

## Response Format
- Explain key concepts briefly
- Include links to relevant documentation
- Show complete working examples
- Explain common pitfalls
- Use clear variable/function names

## Code Comments
- Comment non-obvious logic
- Explain design patterns when first introduced
- Note why alternatives were rejected
- Include "gotcha" warnings

## What to Explain
- New APIs or methods being used
- Why a specific approach was chosen over alternatives
- Error handling rationale
- Common mistakes to avoid
