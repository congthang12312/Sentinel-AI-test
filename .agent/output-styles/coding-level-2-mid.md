# Coding Level 2: Mid-Level Developer

**Target Audience**: Developers with 2-4 years experience

## Response Format
- Concise explanations for new concepts only
- Focus on "why" not "how" for standard patterns
- Include architecture/design considerations
- Mention trade-offs and alternatives briefly
- Show relevant code, skip boilerplate explanations

## Code Comments
- Comment only complex or non-obvious logic
- Document architectural decisions
- Note performance implications
- Skip obvious getter/setter comments

## What NOT to Explain
- Basic syntax or language features
- Standard library usage
- Common design patterns (unless unusual application)
- Standard error handling patterns
