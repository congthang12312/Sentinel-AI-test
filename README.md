# AI-Powered Automation Testing Framework

> AI reads requirements вҶ’ generates BDD Gherkin вҶ’ generates Playwright code вҶ’ executes tests вҶ’ self-heals on failure.

## рҹҸ—пёҸ Architecture

```
                    в”Ңв”Җв”Җв”Җ Step 0: Jira MCP в”Җв”Җв”Җ Fetch requirement from ticket
                    в”Ӯ
Requirements (.md) в”Җв”јв”Җв”Җв”Җ Step 1: Gemini API в”Җв”Җв”Җ Generate Gherkin (.feature)
                    в”Ӯ
                    в”ңв”Җв”Җв”Җ Step 2: Gemini API + Playwright MCP в”Җв”Җв”Җ Generate Page Objects + Steps (.ts)
                    в”Ӯ
                    в”ңв”Җв”Җв”Җ Step 3: Playwright Test в”Җв”Җв”Җ Execute tests
                    в”Ӯ
                    в””в”Җв”Җв”Җ Step 4: Self-Healing в”Җв”Җв”Җ AI auto-fixes broken locators (max 3 retries)
```

**Two independent pipelines:**
- **`make test`** вҖ” Run tests (pure Playwright, no AI, no API key needed)
- **`make ai-gen`** вҖ” AI generates test code from requirements (needs `GEMINI_API_KEY`)

## рҹҡҖ Local Setup

```bash
# 1. Clone and install
git clone <repo-url>
cd mcp-automation-test
npm install

# 2. Install Playwright browsers
npx playwright install chromium

# 3. Create environment file
cp .env.local.example .env.local
# Edit .env.local with your credentials (see Environment Variables below)

# 4. Run all tests
make test

# 5. View Allure report
make report-docker
# вҶ’ http://localhost:5050/allure-docker-service/projects/default/reports/latest/index.html
```

### Quick Install (shortcut)
```bash
make install    # Runs npm install + installs all Playwright browsers
```

### Adding a New Module

```
src/
в”ңв”Җв”Җ features/<module>/<name>.feature     # Gherkin scenarios
в””в”Җв”Җ tests/<module>/
    в”ңв”Җв”Җ steps/<name>.steps.ts            # Step definitions
    в””в”Җв”Җ pages/<name>.page.ts             # Page Object Model
```

1. Create the folder structure under `src/tests/<module>/`
2. Add `.feature` file under `src/features/<module>/`
3. Pages extend `../common/base.page.ts`
4. Add test data in `src/fixtures/<env>/data.json`

## рҹ“ң All Commands

### рҹ§Ә Local Test Execution

| Command | Description |
|---------|-------------|
| `make test` | Run **all** tests locally |
| `make test mod=auth` | Run tests for a specific **module** |
| `make test tag=@smoke` | Run tests matching a **tag** |
| `make test mod=auth tag=@e2e` | Combine module + tag filter |
| `make test-ui` | Run in **Playwright UI** mode |
| `make test-headed` | Run in **headed** browser mode |

### рҹҗі Docker Test Execution

| Command | Description |
|---------|-------------|
| `make docker-test` | Run **ALL** tests inside Docker container |

### рҹ“Ҡ Reports

| Command | Description |
|---------|-------------|
| `make report` | Open HTML Playwright report (local) |
| `make report-docker` | Start **Allure** report server |
| `make stop-report` | Stop all Docker services |

**Allure Dashboard:** `http://localhost:5050/allure-docker-service/projects/default/reports/latest/index.html`

### рҹӨ– AI Pipeline

| Command | Description |
|---------|-------------|
| `make ai-gen req=login-flow.md mod=auth` | Generate test from a **requirement file** |
| `make ai-gen req=PIM-123 mod=pim` | Generate test from a **Jira ticket** (via MCP) |
| `make ai-heal req=login-flow.md` | Run **self-healing** on failing tests |

### рҹ”Қ Code Quality

| Command | Description |
|---------|-------------|
| `make compile` | TypeScript check (src only) |
| `make compile-ai` | TypeScript check (src + ai-agent) |
| `make check` | Prettier + ESLint auto-fix |
| `make clean` | Remove all generated files and reports |

## рҹ”§ Environment Variables

Create `.env.local` with the following:

```env
# в”Җв”Җв”Җ Target Application в”Җв”Җв”Җ
BASE_URL=https://opensource-demo.orangehrmlive.com
SUT_USERNAME=Admin
SUT_PASSWORD=admin123

# в”Җв”Җв”Җ AI Pipeline (optional вҖ” only needed for make ai-gen/ai-heal) в”Җв”Җв”Җ
AI_PROVIDER=gemini
GEMINI_API_KEY=your-gemini-api-key
GEMINI_MODEL=gemini-3.1-flash-lite-preview

# в”Җв”Җв”Җ Jira Integration (optional вҖ” only needed for Jira ticket input) в”Җв”Җв”Җ
JIRA_URL=https://your-site.atlassian.net
JIRA_USERNAME=your-email@example.com
JIRA_API_TOKEN=your-jira-api-token
```

> **Note:** `GEMINI_API_KEY` and `JIRA_*` are only needed for the AI pipeline (`make ai-gen`). Running tests (`make test`) requires only `BASE_URL`, `SUT_USERNAME`, `SUT_PASSWORD`.
